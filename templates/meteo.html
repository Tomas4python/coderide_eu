About the Application

Derived from my learning experiences, this application is designed to harness the Meteo.lt API. This API facilitates the consumption of public data from meteorological stations overseen by the Lithuanian Hydrometeorological Service, a wing under the Ministry of the Environment.

One noteworthy aspect of this application's code structure is its organic evolution (without pre-design). It mirrors my step-by-step learning progression, supplemented by consistent experimentation.

Key Features and Technical Details

  • Modules & Structure: Initially, modules were developed independently, which were eventually unified into a single, cohesive application with a unified GUI.
  • Learning Objectives: My endeavors with this application aimed at mastering API requests, handling diverse file types (.json, .csv, .zip, .txt), and gaining prowess in various libraries.
  • Functionality: The application can fetch both historical and current meteorological data, obtain forecast data from all accessible stations in Lithuania, and offer a comprehensive graphical analysis tool for all acquired data.

In the application's second window, you'll find a table comparing absolute numbers of measured and forecasted data. Meanwhile, the third window offers a visual comparison of this data through a graph. While the table provides a broad overview, the graph highlights the temperature deviations, both positive and negative.
