Conversation Generator (Chatbot)

View code on Github

The main objective of this project was to create a functional and interactive chatbot capable of generating coherent and contextually appropriate responses. The project involves both the technical aspects of machine learning model training and the practical aspects of GUI application development.

About the Model

At the heart of the project is a sequence-to-sequence bidirectional encoder-decoder model built using Keras. This model is designed to generate conversations by encoding the input text and decoding it to produce a response. The encoder processes the input sequence and converts it into a context vector, which is then fed into the decoder to generate the output sequence. Two different response generation methods are implemented: argmax, which selects the most likely next word, and beam search, which considers multiple possible sequences to find the most probable one.

Data preprocessing is a critical part of the model development process. This involves cleaning and preparing the dataset, which consists of dialogue pairs, to ensure the model can learn effectively. The model is trained on this preprocessed data to optimize its ability to generate coherent responses. Once trained, the model is integrated into a real-time chatbot application.

The development paper included in the GitHub repository provides an extensive overview of the project, detailing the steps from data preprocessing to model development and the implementation of the chatbot application. This project aims to showcase the integration of advanced machine learning techniques with practical application development.

Chatbot Application

The application itself, while a smaller part of the project, is crucial for demonstrating the model's capabilities. It features a customtkinter-based GUI with a conversation display field, an input field, and two fields for sample inputs. Users can enter a message and receive a response from the chatbot, or use pre-prepared text samples for quick testing.
