SQLite Database Manager

View code on Github

This project focuses on leveraging SQLite for managing data scraped from media websites, emphasizing my skill in using SQLite databases. The application scrapes two media websites using Selenium and stores the scraped data in SQLite databases. A Tkinter-based GUI allows users to interact with and query these databases via SQL requests, showcasing the practical application of database management in Python.

About the Application

The Tkinter GUI features a menu for scraping commands, where users can perform both shallow and deep scrapes of media websites Epika and Mediateka. The shallow scrape collects a list of movies from the initial search pages, saving this data to a temporary CSV file. The deep scrape utilizes this CSV to gather detailed information from individual movie pages, storing the data in a SQLite database. The process ensures only new movies are added by checking URLs against existing records.

The GUI also includes SQL query entry fields, providing both sample queries and a space for manual entry. The 'Execute' function is designed to handle 'SELECT' queries, fetching data from the databases and displaying it in the results area. This setup allows users to easily retrieve and view data from the SQLite databases.

The results area displays a table of results based on the executed SQL query, efficiently handling large datasets even for broad queries. Additionally, the application includes extensive logging features. Errors are logged in an error.log file, while informational messages, warnings, and errors are streamed to the console. This robust logging system aids in troubleshooting and maintaining the application.
