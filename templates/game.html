Introduction

Shortly after diving into the world of Python through "PYTHON CRASH COURSE, 2ND EDITION" by Eric Matthes, I felt inspired to embark on the journey of game development. What began as an adventure with the `pygame` module soon evolved into a more comprehensive learning experience, leading me to explore the capabilities of the GUI library `tkinter`.

Development Odyssey

In my first weeks of game coding, I aimed for a solid structure. However, as the game evolved into a complex and sophisticated endeavor, I found myself grappling with intricate code — not exactly spaghetti, but certainly more tangled than I would have preferred. As I delved into other app development projects, the value of a clear, cohesive structure became evident. This insight drove me to revisit and largely reengineer the game.

Switching the game's main engine to `tkinter` and `customtkinter` allowed me to achieve more homogeneous code, even if it meant sacrificing some canvas smoothness. This choice was driven by my keen interest in mastering the `tkinter` GUI.

Throughout this process, I deepened my understanding of numerous core libraries, including but not limited to: `tkinter`, `customtkinter`, `PIL`, `os`, `pygame`, `math`, and `random`. Before this, Object-Oriented Programming (OOP) seemed abstract and elusive. Developing ARTILLERY WAR paved my way to a more profound grasp of OOP concepts, from referencing instances and understanding scopes to explore inheritance and function nesting.

Midway, Chat GPT 4.0 came onboard, acting as an invaluable coding companion and a robust learning tool.

Game Overview

ARTILLERY WAR doesn't focus on swift movements; it's an analytical challenge. It's a turn-based strategy where players input results derived from straightforward arithmetic operations—be it azimuth, elevation, wind speed, distance, or direction. With a plethora of calculations to make, it promises to be an intense mental workout.

Features & Architecture:

  • Game Engine: Rooted in `tkinter` and `customtkinter`.
  • Interactive Windows: Four full-screen program windows, each serving unique gameplay interaction, easily navigable via keyboard commands and mouse.
  • Turn-Based Strategy: Tests players on arithmetical calculations to plan their next move.
  • Modular & OOP Structure: Including key modules like `game_loop.py` (game's main loop), `map.py` (battlefield visuals), `radar.py` (tracking & targeting), and many more.
  • Documents: `about.txt` & `rules.txt` provide detailed game insights.
